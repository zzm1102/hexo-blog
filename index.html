JS笔记（三）——构造函数相关

Posted on 2018-12-02

一、面向对象——单例模式

  1. 对象数据类型的作用：

  • 把描述同一个事物（同一个对象）的属性和方法放在一个内存空间下，起到了分组的作用，这样不同事物之间即使属性名相同，相互也不会发生冲突 -> 这种分组编写代码的模式叫做“单例模式”，obj叫做“命名空间”

  • “模块化开发”：根据需求划分不同功能模块，每人负责一部分，同时开发，最后进行代码合并

单例模式虽然解决了分组问题，但不能实现批量生产，属于手工作业模式
工厂模式：把实现同一件事的相同代码放进一个函数中，需要再实现此功能，只需执行函数即可
-> “函数封装” -> “低耦合，高内聚”：减少冗余代码，提高代码重复利用率

继承：子类继承父类中的属性和方法
多态：当前方法的多种形态 -> 后台语言中多态包含重载和重写
重载：方法名相同，参数类型不同。JS中不存在重载，方法名一样，后面的会把前面的覆盖，最后只保留一个
重写：子类重写父类的方法

JS中一个类似重载但不是重载的功能

二、构造函数模式基础

  1. 构造函数模式和工厂函数的区别？
  • 工厂函数：普通函数执行 -> createFn();
  • 构造函数执行 -> var p1 = new CreateFn(); 通过new执行后，CreateFn即为一个类, 函数执行的返回值p1是实例

内置类：创建一个数组：

a. JS中所有的类都是函数数据类型，它通过new执行变成了一个类，但它本身也是一个普通的函数
JS中所有的实例都是对象数据类型
b. 在构造函数模式中，类（函数体）中出现的this.xxx = xxx的this指当前类的一个实例
c. p1 和 p2 都是 CreateJsPerson这个类的实例，都拥有 writeJs 方法，但不同实例之间的方法是不一样的
在类中给实例增加的属性是当前实例的私有属性，实例和实例之间是相互独立的，所以私有属性是不相等的

  1. 在函数代码执行的时候
  • 相同： 都是形成一个私有作用域，然后形参赋值 -> 预解释 -> 代码从上到下执行
  • 不同：在代码执行前，构造函数不用再自己手动创建对象，浏览器会默认创建一个对象数据类型值（这个对象就是我们当前类的实例，用this指代），浏览器再把创建的实例默认返回

构造函数知识点扩展

  1. 在构造函数中，new Fn()执行，如果Fn不需要传递参数，后面的（）可以省略
    // var f1 = new Fn;
  2. this的问题：在类中出现的this.xxx = xxx中的this都是当前类的实例，而某一个属性值（方法）中的this需要看方法执行的时候前面是否有“.” 才能知道this指向谁
  3. 类有普通函数的一面，当函数执行的时候，var num 其实只是当前形成的私有作用域中的私有变量而已，它和f1这个实例没有任何关系，只有this.xxx = xxx才相当于给实例增加私有属性和方法
  4. 在构造函数模式中，浏览器会默认返回实例（一个对象数据类型的值）。）
    如果自己手动return 返回：
    1）返回基本数据类型的值：当前实例不变（return 100; 实例还是f1）
    2）返回引用数据类型的值，当前的f1实例会被返回的值替换
  5. 检测某一个实例是否属于这个类 -> instanceof

    console.log(f1 instanceof Fn); // -> true
    console.log(f1 instanceof Object); // -> true

  6. 检测某一个属性是否属于这个对象（attr in Object），不管是私有还是共有属性，只要存在，用 in 检测都为true。hasOwnProperty：只能用来检测属性是否是对象的“私有属性”

关于内存释放和作用域销毁

  1. 堆内存
    对象数据类型或函数数据类型在定义的时候首先会开辟一个堆内存，堆内存对应一个引用地址，如果外面有变量等知道了这个地址，这个内存就被占用了，就不能销毁。

我们想让堆内存释放/销毁，只需把所有引用它的变量赋值为null即可，浏览器会在空闲的时候把它销毁。

  1. 栈内存
  • 全局作用域
    1）只有当页面关闭的时候才会销毁

  • 私有作用域（只有函数执行会产生私有作用域）
    1）一般情况下，当私有作用域中的代码执行完成后，当前作用域都会主动进行释放和销毁
    2）特殊情况：

    • 当前作用域中的部分内存被作用域以外的东西占用，则当前作用域不能销毁。
      a. 函数执行返回了一个引用数据类型的值，且在函数外面被其他东西接收了 -> 不销毁

      b. 在一个私有作用域中给DOM元素绑定方法，一般情况下私有作用域都不销毁


￼

c. 下述情况*不立即销毁* -> fn 返回的函数没有被其他函数占用，但是还需要执行一次，所以暂时不销毁，待返回的值执行完毕后，
  浏览器会在空闲时候销毁它。

JS中的this关键字

全局：this -> window

JS中的this代表的是当前行为执行的主体；JS中的context代表的是当前行为执行的环境（区域）
this是谁和函数在哪定义或在哪执行都没有关系

如何区分：
  1. 首先看函数前是否有“.”，前面是谁this就指向谁，否则指向window
  2. 自执行函数中的this永远是window
  3. 给元素的某一个事件绑定方法，当事件触发时执行绑定方法，方法中的this是当前元素

在当前的作用域中，JS代码执行之前，浏览器首先会默认把所有带var和function的进行提前声明或者定义

定义：num = 12

  • 2）var => 在预解释的时候只是提前声明，不赋值

    function => 在预解释的时候声明+定义

  • 3）预解释只发生在当前作用域，开始只对window下进行预解释，只有函数执行的时候才对函数内的进行预解释

3. JS中内存的分类

栈内存：用来提供一个供JS代码执行的环境 => 作用域（全局作用域、私有作用域）
堆内存：用来存储引用数据类型的值 => 对象存储的是属性名和属性值，函数存储的是代码字符串

4. 作用域链

1) 区分全局作用域和私有作用域

- 1）在全局作用域下（预解释阶段）声明的变量是全局变量
- 2）在私有作用域中声明的变量和函数的形参都是私有变量
      在私有作用域中，代码执行遇到一个变量，首先确定是否为私有变量，如果是，则与外面的变量没有任何关系
      如果不是，则往当前作用域的上级作用域进行查找，一直找到window为止。（作用域链）

2) 当函数执行的时候，首先会形成一个新的私有作用域，然后按照如下步骤执行：

- 1）如果有形参，先给形参赋值
- 2）进行私有作用域中的预解释
- 3）私有作用域中的代码从上到下依次执行
- 4）。。。。
  • 函数形成私有作用域保护了里面的私有变量不受外界干扰（互相不能修改）=> “闭包” 变量保护机制

3) 在全局作用域中带var和不带var的区别：

  • 带var可以进行预解释，所以在赋值前执行不会报错，不带var会报reference undefined错。
  • num = 12 => 相当于给window增加了一个叫做num的属性名，属性值是12
  • var num = 12 => 相当于给全局作用域增加一个全局变量num，同时也相当于给window增加了一个num的属性名，属性值是12
(1) 预解释的时候，无论条件是否成立，都会把var进行变量提升
(2) 匿名函数之函数表达式：把函数定义的部分当作值赋给变量/元素的事件
e.g.  
fn();  //  => 相当于 undefined()  => error, fn is not a function
var fn = function() {
   console.log(“ok”)
};

预解释为 var fn 还没有赋值

(3) 自执行函数：不进行预解释，当代码执行到这个位置时定义和执行一起完成
(4) 函数体中return后面的代码虽然不再执行了，但还是会进行预解释
(5) 在JS中，如果变量名和函数名重复，也算冲突
window预解释：
