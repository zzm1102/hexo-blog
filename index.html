数据绑定与DOM回流

数据绑定方式：
1.动态创建元素节点并追加到页面中
优势：把需要动态创建的内容一个一个追加到页面中，对原来的元素没有影响
弊端：浏览器每创建一个li就添加到页面中，就会引发一次DOM回流，影响性能

2.字符串拼接（字符串拼接会影响原先已有的元素，但是是模板引擎数据绑定的常用方式）
弊端：把新拼接的字符串添加到#ul中，原有的三个li的标签绑定的事件都消失了
优势：事先把内容拼接好，最后统一添加到页面中，只引发一次回流

DOM回流（重排 reflow）：当页面中的HTML结构发生改变（增加、删除元素，位置发生改变），浏览器都需要重新计算一遍最新的DOM结构，重新对当前页面进行渲染。应尽量减少DOM的回流
重绘：某一个元素的部分样式发生改变（e.g.背景颜色），浏览器只需要重新渲染当前元素即可

3.文档碎片

DOM映射机制：页面中的标签和JS中获取到的元素对象（元素集合）是紧紧绑定在一起的，
页面中的HTML结构改变，JS不需重新获取，集合内的内容也会跟着改变

正则

什么是正则：处理字符串的一种规则 (对象)

  1. 匹配 判断一个字符串是否符合制定的规则 -> reg.test(str)
  2. 捕获 获取符合规则的字符 -> reg.exec(str)

如何创建一个正则

  1. 字面量方式
    var reg = /\d/;
  2. 实例创建方式
    var reg = new RegExp(“”);

不同于创建数组和对象，正则的这两种方法有区别

如何学习

面向对象思想
console.dir(RegExp.prototype);

  1. 具有特殊意义的元字符
    \：转义字符，转义后面字符所代表的含义 \d：0-9的数字
    ^：以某一个元字符开始，不占位
    $：以某一个元字符结尾，不占位 /^\d$/：只能是1个0-9的数字 /^\d\d\d$/：只能是3个数字
    \n：匹配一个换行符
    .：除了\n以外的任意字符
    ()：分组 -> 把一个大正则分成几个小正则
    x|y： x或者y中的一个
    [xyz]：x或y或z中的一个
    [^xyz]：除了xyz三个以外的任何一个字符
    [a-z]：a-z之间的任何一个字符
    [^a-z]：除了a-z之间的任何一个字符
    \d：一个0-9之间的数字
    \D：除了0-9数字之外的任何字符
    \b：匹配一个边界符，不占位 “w1 w2 w3”
    \w：数字、字母、下划线中的任意一个字符 == [0-9a-zA-Z_]
    \s：匹配一个空白字符：空格，制表符、换页符…
  1. 代表出现次数的量词元字符
    *：出现零到多次
    +：出现一到多次（至少出现一次）
    ?：出现零次或一次（出现/不出现）
    {n}：出现n次
    {n,}：出现n到多次
    {n,m}：出现n到m次

函数的三种角色

函数本身会有一些自己的属性：

length: 形参的个数
name: 函数名
prototype: 类的原型，在原型上定义的方法都是当前Fn这个类实例的公有方法
__proto__: 把函数当作一个普通对象，指向Function这个类的原型

一个函数存在多面性：

“普通函数”：它本身就是一个普通函数，执行时形成私有作用域（闭包）、形参赋值、预解释、代码执行、执行完成后栈内存销毁/不销毁

“类”： 它有自己的实例，也有prototype属性 是自己的原型，它的实例都可以指向自己的原型

“普通对象”：和var obj = {} 中的obj一样，就是一个普通的对象，它作为对象可以有自己私有的属性，也可以通过__proto__找到Function.prototype

这三者之间是没有必然关系的

一、原型模式

基于构造函数模式的原型模式解决了方法或者属性公有的问题 -> 把实例之间相同的属性和方法提取成公有的属性和方法（xxxprototype.xxx）

  1. 每一个函数数据类型（普通函数、类）都有一个自带的属性：prototype，并且这个属性是一个对象数据类型的值
  2. 在prototype上类蓝旗给它加了一个属性constructor（构造函数），属性值是当前函数（类）本身
  3. 每一个对象数据类型（普通对象、实例、prototype…）也再带一个属性proto，属性值是当前实例所属类的原型（prototype）

Object是JS中所有对象数据类型的基类（最顶层的类）

1）f1 instanceof Object -> true 因为f1通过proto可以向上级查找，最后总能找到Object
2）在Object.prototype上没有proto这个属性

原型链模式

通过 对象名.属性名 的方式获取属性值的时候，首先在对象私有属性上进行查找，如果私有属性存在这个值，则获取私有属性值。
如果私有的没有，则通过proto找到所属类的原型（类的原型上定义的属性和方法都是当前实例公有的属性和方法），一直找到Object.prototype为止。

IE浏览器中原型模式也是同样原理，但由于IE担心通过proto把公有属性修改，因此禁止使用proto

二、批量设置原型上的公有属性和方法

  1. 起别名：pro = Fn.prototype; 把原来原型指向的地址赋值给pro，从而操作同一个内存空间
  2. 重构原型对象方式

// 只有浏览器天生给Fn.prototype开辟的堆内存才有constructor，自己开辟的没有，constructor不再是指向Fn，而是Object
console.log(f.constructor); // -> Object (没做处理前)

用这种方式给内置类增加公有属性会替换掉原来已经存在的原型和方法，因此会被浏览器屏蔽
因此只能单独修改内置类的方法，如果方法名重复，会修改内置方法 -> 在内置类的原型上加方法，都需要加特殊前缀

三、深入扩展原型链模式常用的六种继承方式

Object.create()方法创建一个拥有指定原型和若干个指定属性的新对象，使用现有的对象来提供新创建的对象的proto。
Object.create(proto, [propertiesObject])

1. “原型继承”是JS中最常用的一种继承方式

子类B想继承父类A的所有属性和方法（私有+公有），只需要 B.prototype = new A 即可。
原型继承的特点：把父类中私有+公有都继承到子类原型上（子类公有的）（父类私有+公有 -> 子类公有）

3. 冒充对象继承

把父类私有+公有属性和方法克隆一份作为子类私有的属性和方法（父类私有+公有 -> 子类私有）

4. 混合模式继承

原型继承 + call 继承 (父类私有 -> 子类私有 && 父类公有+私有 -> 子类公有)

JS笔记（三）——构造函数相关

Posted on 2018-12-02

一、面向对象——单例模式

  1. 对象数据类型的作用：

  • 把描述同一个事物（同一个对象）的属性和方法放在一个内存空间下，起到了分组的作用，这样不同事物之间即使属性名相同，相互也不会发生冲突 -> 这种分组编写代码的模式叫做“单例模式”，obj叫做“命名空间”

  • “模块化开发”：根据需求划分不同功能模块，每人负责一部分，同时开发，最后进行代码合并

单例模式虽然解决了分组问题，但不能实现批量生产，属于手工作业模式
工厂模式：把实现同一件事的相同代码放进一个函数中，需要再实现此功能，只需执行函数即可
-> “函数封装” -> “低耦合，高内聚”：减少冗余代码，提高代码重复利用率

继承：子类继承父类中的属性和方法
多态：当前方法的多种形态 -> 后台语言中多态包含重载和重写
重载：方法名相同，参数类型不同。JS中不存在重载，方法名一样，后面的会把前面的覆盖，最后只保留一个
重写：子类重写父类的方法

JS中一个类似重载但不是重载的功能

二、构造函数模式基础

  1. 构造函数模式和工厂函数的区别？
  • 工厂函数：普通函数执行 -> createFn();
  • 构造函数执行 -> var p1 = new CreateFn(); 通过new执行后，CreateFn即为一个类, 函数执行的返回值p1是实例

内置类：创建一个数组：

a. JS中所有的类都是函数数据类型，它通过new执行变成了一个类，但它本身也是一个普通的函数
JS中所有的实例都是对象数据类型
b. 在构造函数模式中，类（函数体）中出现的this.xxx = xxx的this指当前类的一个实例
c. p1 和 p2 都是 CreateJsPerson这个类的实例，都拥有 writeJs 方法，但不同实例之间的方法是不一样的
在类中给实例增加的属性是当前实例的私有属性，实例和实例之间是相互独立的，所以私有属性是不相等的

  1. 在函数代码执行的时候
  • 相同： 都是形成一个私有作用域，然后形参赋值 -> 预解释 -> 代码从上到下执行
  • 不同：在代码执行前，构造函数不用再自己手动创建对象，浏览器会默认创建一个对象数据类型值（这个对象就是我们当前类的实例，用this指代），浏览器再把创建的实例默认返回

构造函数知识点扩展

  1. 在构造函数中，new Fn()执行，如果Fn不需要传递参数，后面的（）可以省略
    // var f1 = new Fn;
  2. this的问题：在类中出现的this.xxx = xxx中的this都是当前类的实例，而某一个属性值（方法）中的this需要看方法执行的时候前面是否有“.” 才能知道this指向谁
  3. 类有普通函数的一面，当函数执行的时候，var num 其实只是当前形成的私有作用域中的私有变量而已，它和f1这个实例没有任何关系，只有this.xxx = xxx才相当于给实例增加私有属性和方法
  4. 在构造函数模式中，浏览器会默认返回实例（一个对象数据类型的值）。）
    如果自己手动return 返回：
    1）返回基本数据类型的值：当前实例不变（return 100; 实例还是f1）
    2）返回引用数据类型的值，当前的f1实例会被返回的值替换
  5. 检测某一个实例是否属于这个类 -> instanceof

    console.log(f1 instanceof Object); // -> true

  6. 检测某一个属性是否属于这个对象（attr in Object），不管是私有还是共有属性，只要存在，用 in 检测都为true。hasOwnProperty：只能用来检测属性是否是对象的“私有属性”

关于内存释放和作用域销毁

  1. 堆内存
    对象数据类型或函数数据类型在定义的时候首先会开辟一个堆内存，堆内存对应一个引用地址，如果外面有变量等知道了这个地址，这个内存就被占用了，就不能销毁。

我们想让堆内存释放/销毁，只需把所有引用它的变量赋值为null即可，浏览器会在空闲的时候把它销毁。

  1. 栈内存
  • 全局作用域
    1）只有当页面关闭的时候才会销毁

  • 私有作用域（只有函数执行会产生私有作用域）
    1）一般情况下，当私有作用域中的代码执行完成后，当前作用域都会主动进行释放和销毁
    2）特殊情况：

    • 当前作用域中的部分内存被作用域以外的东西占用，则当前作用域不能销毁。
      a. 函数执行返回了一个引用数据类型的值，且在函数外面被其他东西接收了 -> 不销毁

      b. 在一个私有作用域中给DOM元素绑定方法，一般情况下私有作用域都不销毁


￼

c. 下述情况*不立即销毁* -> fn 返回的函数没有被其他函数占用，但是还需要执行一次，所以暂时不销毁，待返回的值执行完毕后，
  浏览器会在空闲时候销毁它。

JS中的this关键字

全局：this -> window

JS中的this代表的是当前行为执行的主体；JS中的context代表的是当前行为执行的环境（区域）
this是谁和函数在哪定义或在哪执行都没有关系

如何区分：
  1. 自执行函数中的this永远是window，“use struct” 严格模式下，自执行函数的this永远是undefined
  2. 首先看函数前是否有“.”，前面是谁this就指向谁，否则指向window（严格模式下是undefined）
  3. 给元素的某一个事件绑定方法，当事件触发时执行绑定方法，方法中的this是当前元素
  4. 在构造函数模式中，类（函数体）中出现的this.xxx = xxx的this指当前类的一个实例
  5. 可以使用call、apply、bind改变this的指向

3. JS中内存的分类

栈内存：用来提供一个供JS代码执行的环境 => 作用域（全局作用域、私有作用域）

4. 作用域链

1) 区分全局作用域和私有作用域

- 1）在全局作用域下（预解释阶段）声明的变量是全局变量
- 2）在私有作用域中声明的变量和函数的形参都是私有变量
      在私有作用域中，代码执行遇到一个变量，首先确定是否为私有变量，如果是，则与外面的变量没有任何关系
      如果不是，则往当前作用域的上级作用域进行查找，一直找到window为止。（作用域链）

2) 当函数执行的时候，首先会形成一个新的私有作用域，然后按照如下步骤执行：

- 1）如果有形参，先给形参赋值
- 2）进行私有作用域中的预解释
- 3）私有作用域中的代码从上到下依次执行
- 4）。。。。
  • 函数形成私有作用域保护了里面的私有变量不受外界干扰（互相不能修改）=> “闭包” 变量保护机制

3) 在全局作用域中带var和不带var的区别：

  • 带var可以进行预解释，所以在赋值前执行不会报错，不带var会报reference undefined错。
  • num = 12 => 相当于给window增加了一个叫做num的属性名，属性值是12
  • var num = 12 => 相当于给全局作用域增加一个全局变量num，同时也相当于给window增加了一个num的属性名，属性值是12
(1) 预解释的时候，无论条件是否成立，都会把var进行变量提升

(4) 函数体中return后面的代码虽然不再执行了，但还是会进行预解释
