JS笔记（一）——数据类型、预解释

Posted on 2018-10-26 | In Notes

Js数据类型

基本数据类型：
Number、String、Boolean、null、undefined
引用数据类型：
Object：{}、[]、/$^/、Date
Function

1. 当浏览器加载HTML页面的时候，首先会提供一个供全局JS代码执行的环境 -> 全局作用域（global/window）

2. 预解释（变量提升）

在当前的作用域中，JS代码执行之前，浏览器首先会默认把所有带var和function的进行提前声明或者定义

  • 1）理解声明和定义

    声明：var num;   如果一个只是声明了，但没有赋值，默认的值是undefined
定义：num = 12

  • 2）var => 在预解释的时候只是提前声明，不赋值

    function => 在预解释的时候声明+定义

    e.g.

  • 3）预解释只发生在当前作用域，开始只对window下进行预解释，只有函数执行的时候才对函数内的进行预解释

3. JS中内存的分类

栈内存：用来提供一个供JS代码执行的环境 => 作用域（全局作用域、私有作用域）
堆内存：用来存储引用数据类型的值 => 对象存储的是属性名和属性值，函数存储的是代码字符串

4. 作用域链

1) 区分全局作用域和私有作用域

- 1）在全局作用域下（预解释阶段）声明的变量是全局变量
- 2）在私有作用域中声明的变量和函数的形参都是私有变量
      在私有作用域中，代码执行遇到一个变量，首先确定是否为私有变量，如果是，则与外面的变量没有任何关系
      如果不是，则往当前作用域的上级作用域进行查找，一直找到window为止。（作用域链）

2) 当函数执行的时候，首先会形成一个新的私有作用域，然后按照如下步骤执行：

- 1）如果有形参，先给形参赋值
- 2）进行私有作用域中的预解释
- 3）私有作用域中的代码从上到下依次执行
- 4）。。。。
  • 函数形成私有作用域保护了里面的私有变量不受外界干扰（互相不能修改）=> “闭包” 变量保护机制

3) 在全局作用域中带var和不带var的区别：

  • 带var可以进行预解释，所以在赋值前执行不会报错，不带var会报reference undefined错。
  • num = 12 => 相当于给window增加了一个叫做num的属性名，属性值是12
  • var num = 12 => 相当于给全局作用域增加一个全局变量num，同时也相当于给window增加了一个num的属性名，属性值是12
(1) 预解释的时候，无论条件是否成立，都会把var进行变量提升
(2) 匿名函数之函数表达式：把函数定义的部分当作值赋给变量/元素的事件
e.g.  
fn();  //  => 相当于 undefined()  => error, fn is not a function
var fn = function() {
   console.log(“ok”)
};

预解释为 var fn 还没有赋值

(3) 自执行函数：不进行预解释，当代码执行到这个位置时定义和执行一起完成
(4) 函数体中return后面的代码虽然不再执行了，但还是会进行预解释
(5) 在JS中，如果变量名和函数名重复，也算冲突
window预解释：
// 声明+定义： fn = xxxfff111;
// 声明 var fn; (不需要重新声明)
// 声明（不重复进行）+ 定义 fn = xxxfff222
// => fn = xxxfff222;

5. 如何查找上级作用域

看当前函数是在哪个作用域下定义的，那么它的上级作用域就是谁
和函数在哪执行没有关系
