JS笔记（四）——原型链模式

Posted on 2018-12-04

一、原型模式

基于构造函数模式的原型模式解决了方法或者属性公有的问题 -> 把实例之间相同的属性和方法提取成公有的属性和方法（xxxprototype.xxx）

  1. 每一个函数数据类型（普通函数、类）都有一个自带的属性：prototype，并且这个属性是一个对象数据类型的值
  2. 在prototype上类蓝旗给它加了一个属性constructor（构造函数），属性值是当前函数（类）本身
  3. 每一个对象数据类型（普通对象、实例、prototype…）也再带一个属性proto，属性值是当前实例所属类的原型（prototype）

Object是JS中所有对象数据类型的基类（最顶层的类）

1）f1 instanceof Object -> true 因为f1通过proto可以向上级查找，最后总能找到Object
2）在Object.prototype上没有proto这个属性

原型链模式

通过 对象名.属性名 的方式获取属性值的时候，首先在对象私有属性上进行查找，如果私有属性存在这个值，则获取私有属性值。
如果私有的没有，则通过proto找到所属类的原型（类的原型上定义的属性和方法都是当前实例公有的属性和方法），一直找到Object.prototype为止。

IE浏览器中原型模式也是同样原理，但由于IE担心通过proto把公有属性修改，因此禁止使用proto

二、批量设置原型上的公有属性和方法

  1. 起别名：pro = Fn.prototype; 把原来原型指向的地址赋值给pro，从而操作同一个内存空间
  2. 重构原型对象方式

// 只有浏览器天生给Fn.prototype开辟的堆内存才有constructor，自己开辟的没有，constructor不再是指向Fn，而是Object
console.log(f.constructor); // -> Object (没做处理前)

用这种方式给内置类增加公有属性会替换掉原来已经存在的原型和方法，因此会被浏览器屏蔽
因此只能单独修改内置类的方法，如果方法名重复，会修改内置方法 -> 在内置类的原型上加方法，都需要加特殊前缀

三、深入扩展原型链模式常用的六种继承方式

Object.create()方法创建一个拥有指定原型和若干个指定属性的新对象，使用现有的对象来提供新创建的对象的proto。
Object.create(proto, [propertiesObject])
￼

1. “原型继承”是JS中最常用的一种继承方式

子类B想继承父类A的所有属性和方法（私有+公有），只需要 B.prototype = new A 即可。
原型继承的特点：把父类中私有+公有都继承到子类原型上（子类公有的）（父类私有+公有 -> 子类公有）

2.call 继承

把父类私有的属性和方法克隆一份作为子类私有的属性和方法 （父类私有 -> 子类私有）

3. 冒充对象继承

把父类私有+公有属性和方法克隆一份作为子类私有的属性和方法（父类私有+公有 -> 子类私有）

4. 混合模式继承

原型继承 + call 继承 (父类私有 -> 子类私有 && 父类公有+私有 -> 子类公有)

6. 中间类继承法 -> 不兼容, 可用于移动端
