JS笔记（七）——正则

Posted on 2018-12-10 | In Notes

什么是正则：处理字符串的一种规则 (对象)

  1. 匹配 判断一个字符串是否符合制定的规则 -> reg.test(str)
  2. 捕获 获取符合规则的字符 -> reg.exec(str)

如何创建一个正则

  1. 字面量方式
    var reg = /\d/;
  2. 实例创建方式
    var reg = new RegExp(“”);

不同于创建数组和对象，正则的这两种方法有区别
1）字面量方式中出现的一切都是元字符，不能进行变量拼接，实例方式可以
2）字面量方式中直接写\d即可，实例中需要\d 转义

如何学习

面向对象思想
console.dir(RegExp.prototype);

  1. 具有特殊意义的元字符
    \：转义字符，转义后面字符所代表的含义 \d：0-9的数字
    ^：以某一个元字符开始，不占位
    $：以某一个元字符结尾，不占位 /^\d$/：只能是1个0-9的数字 /^\d\d\d$/：只能是3个数字
    \n：匹配一个换行符
    .：除了\n以外的任意字符
    ()：分组 -> 把一个大正则分成几个小正则
    x|y： x或者y中的一个
    [xyz]：x或y或z中的一个
    [^xyz]：除了xyz三个以外的任何一个字符
    [a-z]：a-z之间的任何一个字符
    [^a-z]：除了a-z之间的任何一个字符
    \d：一个0-9之间的数字 [0-9]
    \D：除了0-9数字之外的任何字符
    \b：匹配一个边界符，不占位 “w1 w2 w3”
    \w：数字、字母、下划线中的任意一个字符 == [0-9a-zA-Z_]
    \s：匹配一个空白字符：空格，制表符、换页符…

  1. 代表出现次数的量词元字符
    *：出现零到多次
    +：出现一到多次（至少出现一次）
    ?：出现零次或一次（出现/不出现）
    {n}：出现n次
    {n,}：出现n到多次
    {n,m}：出现n到m次

  2. 修饰符：g, i, m
    g (global): 全局匹配
    i (ignoreCase): 忽略大小写
    m (multiline): 多行匹配

规律：
  1. []中出现的字符都是本身，不代表特殊含义
  2. []不识别两位数 /^[12]$/ -> 1or2 /^[12-68]$/ ->1、2-6中的一个、8 这三者中的一个
    var reg = /^[.]$/
    reg.test(“1”); // false
    reg.test(“.”); // true
  3. | 或的优先级很低，需要()改变优先级
    var reg = /^18|19$/; // 18、19、181、189、119、819…
    var reg = /^(18|19)$/; // 18、19
  4. 分组的作用：改变优先级、分组引用

exec -> 正则的捕获

捕获的内容格式

每次捕获先进行默认匹配，没有匹配成功结果为null，只有匹配的内容才能捕捉到

  1. 捕获的内容是一个数组，数组第一项是当前大正则捕获的内容
    index：捕获内容在字符串中开始的索引位置
    input：捕获的原始字符串

  2. 正则捕获的特点
    1) 懒惰性 -> 每次执行只捕获第一个匹配内容，不进行处理的情况下执行多次捕获，结果还是第一个匹配内容
    lastIndex: 正则每次捕获在字符串中开始查找的位置，默认值0
    2）如何解决懒惰性：在正则末尾加修饰符“g”
    原理：加了全局修饰符g，正则每次捕获后lastIndex值变为最新的值，下次捕获从最新位置开始查找
    3）正则的贪婪性：每次捕获匹配最长的结果 e.g. 1和18都符合正则，捕获的是18
    4）如何解决贪婪性 -> 在量词元字符后加“?” var reg = /\d+?/g;

?在正则中的作用
普通元字符后：出现0-1次 /\d?/ -> 数字可能出现也可能不出现
量词元字符后：取消捕获的贪婪性

  1. 字符串中的match方法 -> 吧所有和正则匹配的字符串都捕获

    分组捕获时不好用，无法捕获小正则内容
    (?:)：在分组中，?:代表只匹配不捕获

正则分组

  1. 改变优先级
  2. 分组引用
    \1代表和第二个分组一样的内容，\2代表和第二个分组一样的内容
  3. 分组捕获

replace:把原有的字符替换成新的字符

在不使用正则的情况下，每执行一次只能替换一个字符

正则练习：

  1. 替换

  2. 获取一个字符串中出现次数最多的字符，并且获取出现次数

  3. 模板引擎实现的初步原理

  4. 获取url中的参数，并保存成对象格式

  5. 时间字符串格式化

2.无分组符 () 时，对于其他限定字符，单个描述为一组，对于 | 往前截止到开头或左括号 (，往后截止到结尾或右括号 ) 为一组。举例：

对于限定字符,正则表达式 /abc/ 中 限定的仅为字母c, 因此匹配的字符串为 ab/abc/abccc 等，不匹配 abcabc;

对于多选结构，正则表达式 /abc|def/ 匹配的是字符串 abc 或 def,不匹配 abcef/abdef。

更复杂一点的例子，正则表达式 /^\d{2}|\d{4}[\dX]$/ 的可以匹配 以两个数字开头 和 以四个数字+一个数字或字母X结尾 的字符串。
