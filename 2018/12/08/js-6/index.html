JS笔记（六）——数据绑定和DOM结构、正则

Posted on 2018-12-08

数据绑定与DOM回流

数据绑定方式：
1.动态创建元素节点并追加到页面中
优势：把需要动态创建的内容一个一个追加到页面中，对原来的元素没有影响
弊端：浏览器每创建一个li就添加到页面中，就会引发一次DOM回流，影响性能

2.字符串拼接（字符串拼接会影响原先已有的元素，但是是模板引擎数据绑定的常用方式）
弊端：把新拼接的字符串添加到#ul中，原有的三个li的标签绑定的事件都消失了
优势：事先把内容拼接好，最后统一添加到页面中，只引发一次回流

DOM回流（重排 reflow）：当页面中的HTML结构发生改变（增加、删除元素，位置发生改变），浏览器都需要重新计算一遍最新的DOM结构，重新对当前页面进行渲染。应尽量减少DOM的回流
重绘：某一个元素的部分样式发生改变（e.g.背景颜色），浏览器只需要重新渲染当前元素即可

3.文档碎片

DOM映射机制：页面中的标签和JS中获取到的元素对象（元素集合）是紧紧绑定在一起的，
页面中的HTML结构改变，JS不需重新获取，集合内的内容也会跟着改变

正则

什么是正则：处理字符串的一种规则 (对象)

  1. 匹配 判断一个字符串是否符合制定的规则 -> reg.test(str)
  2. 捕获 获取符合规则的字符 -> reg.exec(str)

如何创建一个正则

  1. 字面量方式
    var reg = /\d/;
  2. 实例创建方式
    var reg = new RegExp(“”);

不同于创建数组和对象，正则的这两种方法有区别

如何学习

面向对象思想
console.dir(RegExp.prototype);

  1. 具有特殊意义的元字符
    \：转义字符，转义后面字符所代表的含义 \d：0-9的数字
    ^：以某一个元字符开始，不占位
    $：以某一个元字符结尾，不占位 /^\d$/：只能是1个0-9的数字 /^\d\d\d$/：只能是3个数字
    \n：匹配一个换行符
    .：除了\n以外的任意字符
    ()：分组 -> 把一个大正则分成几个小正则
    x|y： x或者y中的一个
    [xyz]：x或y或z中的一个
    [^xyz]：除了xyz三个以外的任何一个字符
    [a-z]：a-z之间的任何一个字符
    [^a-z]：除了a-z之间的任何一个字符
    \d：一个0-9之间的数字 [0-9]
    \D：除了0-9数字之外的任何字符
    \b：匹配一个边界符，不占位 “w1 w2 w3”
    \w：数字、字母、下划线中的任意一个字符 == [0-9a-zA-Z_]
    \s：匹配一个空白字符：空格，制表符、换页符…
  1. 代表出现次数的量词元字符
    *：出现零到多次
    +：出现一到多次（至少出现一次）
    ?：出现零次或一次（出现/不出现）
    {n}：出现n次
    {n,}：出现n到多次
    {n,m}：出现n到m次
规律：
  1. []中出现的字符都是本身，不代表特殊含义
    var reg = /^[.]$/
    reg.test(“1”); // false
    reg.test(“.”); // true
  2. | 或的优先级很低，需要()改变优先级
    var reg = /^18|19$/; // 18、19、181、189、119、819…
    var reg = /^(18|19)$/; // 18、19
  3. 分组的作用：改变优先级
